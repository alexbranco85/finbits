- Valida se: Datas não são futuras
- Valida se: Descrição tem no mínimo 3 caracteres

Adicionadas validações condicionais customizadas no transactionItemSchema usando
.refine() do Zod baseadas na description.

NOVA FUNÇÃO UTILITÁRIA (src/schemas/transactionSchema.ts):
  export function isPrime(num: number): boolean {
    if (num <= 1) return false;
    if (num <= 3) return true;
    if (num % 2 === 0 || num % 3 === 0) return false;
    for (let i = 5; i * i <= num; i += 6) {
      if (num % i === 0 || num % (i + 2) === 0) return false;
    }
    return true;
  }

VALIDAÇÕES ADICIONADAS AO transactionItemSchema:

1. INVESTIMENT - valor > 1000:
   .refine((data) => {
     if (data.description === 'Investiment') {
       return data.value > 1000;
     }
     return true;
   }, {
     message: 'Para description "Investiment", o valor deve ser maior que 1000',
     path: ['value'],
   })

2. STOCK BUY - valor divisível por 3:
   .refine((data) => {
     if (data.description === 'Stock buy') {
       return data.value % 3 === 0;
     }
     return true;
   }, {
     message: 'Para description "Stock buy", o valor deve ser divisível por 3',
     path: ['value'],
   })

3. STOCK BUY - data d+1 (amanhã ou futura):
   .refine((data) => {
     if (data.description === 'Stock buy') {
       const date = parseDate(data.dtTransaction);
       if (date === null) return true;
       const tomorrow = new Date(todayUTC());
       tomorrow.setUTCDate(tomorrow.getUTCDate() + 1);
       return date >= tomorrow;
     }
     return true;
   }, {
     message: 'Para description "Stock buy", a data deve ser d+1 (amanhã ou futura)',
     path: ['dtTransaction'],
   })

4. ICPA - valor não pode ser primo:
   .refine((data) => {
     if (data.description === 'ICPA') {
       return !isPrime(data.value);
     }
     return true;
   }, {
     message: 'Para description "ICPA", o valor não pode ser um número primo',
     path: ['value'],
   })

AJUSTE NA VALIDAÇÃO DE DATA:
A validação original não permitia datas futuras para nenhuma description.
Foi modificada para permitir datas futuras especificamente para "Stock buy":

  .refine((data) => {
    // Para Stock buy, a data deve ser futura (d+1 ou posterior)
    if (data.description === 'Stock buy') {
      return true; // validação específica abaixo
    }
    // Para outras descriptions, data não pode ser futura
    const date = parseDate(data.dtTransaction);
    return date !== null && date <= todayUTC();
  }, {
    message: 'dtTransaction não pode ser uma data futura',
    path: ['dtTransaction'],
  })
